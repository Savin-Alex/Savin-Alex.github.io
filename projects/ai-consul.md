# AI Consul

[RU](./ai-consul.md) · [EN](../en/projects/ai-consul.md)

Privacy-first AI-ассистент для живых разговоров — интервью, встреч и учёбы.

![AI Consul overview](../docs/images/hero-overview.svg)

Десктоп-приложение, которое показывает транскрипт и AI-подсказки прямо по ходу разговора и при этом оставляет за пользователем контроль над приватностью, поведением при сбоях и режимами обработки звука.

## Паспорт проекта

| Поле | Содержание |
|---|---|
| **Проблема** | Голосовые ассистенты по умолчанию отправляют звук в облако, скрывают свои сбои и почти не дают пользователю прозрачности и контроля над обработкой данных. Вдобавок транскрибация в реальном времени без тонкой настройки тормозит и галлюцинирует. |
| **Цель** | Privacy-first десктоп-ассистент с транскриптом в реальном времени и AI-подсказками для интервью, встреч и учёбы: по умолчанию работает на устройстве, с понятными настройками приватности и поведения при сбоях, а статус движка, стоимость и обращения к сети всегда на виду. |
| **В scope** | Многооконный интерфейс (сессия · транскрипт · companion-оверлей поверх всех окон); локальный STT в реальном времени через WhisperLiveKit за fallback-роутером из 5 движков; локальный LLM (Ollama) с облачным fallback (OpenAI/Anthropic/Google); опора на резюме и описание вакансии; архив сессий и локальный разбор; приватность, стоимость и статус движка — в интерфейсе. |
| **Вне scope** | Упаковка для production, подпись, защита обновлений; захват системного звука на Windows; формальный security review и политика хранения данных и согласия; проверка на реальном многоязычном аудио; бенчмарки в разных условиях. |

**Критерии успеха (со статусом):**
- Local-first по умолчанию — транскрибация и LLM на устройстве, облако опционально → **выполнено**
- Ограниченная задержка в реальном времени — в худшем случае < ~1 с, без зависаний декодера → **выполнено** (23 с → <1 с; 15 → 0 watchdog-сбросов)
- Privacy-инвариант — байты документа не пересекают границу main→renderer → **выполнено** (только booleans)
- Прозрачные сбои — статус движка, fallback и стоимость на виду → **выполнено**
- Контроль качества — автотесты и release-гейты по задержке в худшем случае и доле спайков → **выполнено** (638 тестов)
- Production-ready — подпись, мультиплатформенность, security review, многоязычная проверка, формальные бенчмарки → **не выполнено** (осознанно вне scope)

## Продуктовые решения

- Многооконный интерфейс вместо одного окна чата.
- Local-first по умолчанию, с явными переключателями на гибрид и облако.
- Статус движка и понимание стоимости — часть продукта.
- Fallback — управляемый сценарий, а не скрытый сбой.
- Отдельные окна транскрипта и companion для разговоров, где важна концентрация.

## Технические достижения

- **A/B потокового декодирования (с замерами).** Собрал воспроизводимый A/B-харнесс, чтобы сравнить два алгоритма потокового декодирования Whisper — LocalAgreement-2 и AlignAtt/SimulStreaming (2 парные сессии). Переключение алгоритма по умолчанию на AlignAtt снизило задержку коммита в худшем случае с ~23 с до <1 с, а зависания декодера (15 → 0 watchdog-сбросов) и галлюцинации (16 → 0) исчезли за одну сессию. Числа записаны в логи и воспроизводятся скриптом A/B-сравнения.
- **Диагностика холодного старта.** Проследил галлюцинации первой сессии — спайк задержки ~20 с и мусорный транскрипт на 1476 символов — до прогрева модели: он зависел от сетевого ресурса и молча отваливался по таймауту. Починил, упаковав прогрев локально.
- **Честные метрики задержки.** Заметил, что перцентили задержки схлопываются в ноль из-за того, как новый режим формирует коммиты, — и средние прячут реальные спайки. Вместо них добавил проверки задержки в худшем случае и доли спайков как pre-ship-гейты.

## Метрики (snapshot)

Замеры на парных ~1-часовых сессиях, одна машина (Apple Silicon, small.en); внутренний A/B, а не бенчмарк в разных условиях. Полный отчёт: [A/B benchmark](/evidence/consul-ab-benchmark.html).

| Метрика | Результат |
|---|---|
| Задержка коммита STT в худшем случае (A/B алгоритмов) | 23 с → <1 с |
| Зависания декодера (watchdog resets) / сессия | 15 → 0 |
| Отброшенные галлюцинации (коммиты) / сессия | 16 → 0 |
| Ошибки пайплайна / сессия | 24 → 0 |
| Спайк задержки на холодном старте (фикс прогрева) | 20.2 с → устранён |
| STT-движков за одним fallback-роутером | 5 (WhisperLiveKit, whisper.cpp, Apple Speech, Deepgram, AssemblyAI) |
| LLM-провайдеров (1 локальный + 3 облачных fallback) | 4 (Ollama · OpenAI · Anthropic · Google) |
| Автотестов | 638 проходят |
| Байт документа через IPC main→renderer | 0 (только booleans — privacy-инвариант) |

Один риск реализовался (R1), один снижен (R2), два открыты (R3–R4) — честно о том, что ещё не доказано. R1 и строка про холодный старт в метриках — одна история: риск выявили, он реализовался, ущерб измерили, риск закрыли.

## Ретро / Lessons Learned

- **Что сработало:** относиться к производительности как к эксперименту. Воспроизводимый A/B-харнесс и инструментирование каждого коммита превратили расплывчатое «тормозит» в измеренный выигрыш по задержке в худшем случае (с ~23 с до <1 с) — смена алгоритма стала решением по данным, а не догадкой.
- **Что не сработало:** заглушки и зависящие от среды куски, которые падали молча. Stub-суммаризатор привязывал подсказки к устаревшему контексту, а прогрев модели зависел от сетевого запроса и отваливался по таймауту у пользователя (~20 с задержки в первой сессии и галлюцинации).
- **Что бы изменил:** инструментировать задержку и упаковывать/проверять runtime-зависимости с первого дня; проверять методологию метрик, прежде чем ей доверять — перцентили показывали «0 мс» и прятали реальные спайки, пока я не добавил гейты по монотонному худшему случаю и доле спайков.

## Статус

- Репозиторий приватный.
- В публичном кейсе доступны интерактивное демо, продуктовое позиционирование и измеренный A/B-бенчмарк.
- Перед production-релизом нужны более широкие формальные бенчмарки, security review, усиление упаковки и более широкое тестирование с пользователями.
