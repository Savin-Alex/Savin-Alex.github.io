# AI Consul

[RU](./ai-consul.md) · [EN](../en/projects/ai-consul.md)

Privacy-first AI-ассистент для live-разговоров, интервью, встреч и учебных сессий.

![AI Consul overview](../docs/images/hero-overview.svg)

Desktop-продукт, который даёт пользователю live-транскрипт и AI-подсказки прямо во время разговора, сохраняя контроль над privacy, fallback-поведением и режимами обработки аудио.

## Паспорт проекта

| Поле | Содержание |
|---|---|
| **Проблема** | Conversation-ассистенты по умолчанию стримят аудио в облако, прячут свои failure modes и почти не дают пользователю видимости и контроля над обработкой данных. Плюс real-time транскрибация без тонкой настройки тормозит и галлюцинирует. |
| **Цель** | Privacy-first десктоп-ассистент с live-транскриптом и AI-подсказками для интервью, встреч и учебных сессий — по умолчанию на устройстве, с явными privacy/fallback-контролами и видимыми статусом движка, стоимостью и сетью. |
| **В scope** | Multi-window desktop UX (сессия · транскрипт · always-on-top companion-оверлей); real-time локальный STT через WhisperLiveKit за fallback-роутером из 5 движков; локальный LLM (Ollama) с облачным fallback (OpenAI/Anthropic/Google); grounding по резюме/JD; архив сессий + локальный разбор; privacy/стоимость/статус движка в интерфейсе. |
| **Вне scope** | Production-упаковка/подпись/update-hardening; нативный системный звук на Windows; формальный security review + data-retention/consent; мультиязычная валидация на реальном аудио; multi-condition бенчмарки. |

**Критерии успеха (со статусом):**
- Local-first по умолчанию — транскрибация + LLM на устройстве, облако опционально → **выполнено**
- Ограниченная real-time задержка — worst-case < ~1 с, без зависаний декодера → **выполнено** (23 с → <1 с; 15 → 0 watchdog-сбросов)
- Privacy-инвариант — байты документа не пересекают границу main→renderer → **выполнено** (только booleans)
- Прозрачные failure modes — статус движка, fallback и стоимость видны → **выполнено**
- Контроль качества — автотесты + release-гейты по worst-case-задержке/доле спайков → **выполнено** (638 тестов)
- Production-ready — подпись, мультиплатформенность, security review, мультиязычная валидация, формальные бенчмарки → **не выполнено** (осознанно вне scope)

## Продуктовые решения

- Multi-window desktop UX вместо single chat interface.
- Local-first по умолчанию с явными hybrid/cloud контролами.
- Статус движка и cost awareness как часть продукта.
- Fallback как управляемый сценарий, а не скрытая техническая ошибка.
- Отдельные поверхности транскрипта и companion для разговоров с высокой концентрацией.

## Технические достижения

- **A/B потокового декодирования (с замерами).** Собрал воспроизводимый A/B-харнесс для сравнения двух алгоритмов real-time декодирования Whisper — LocalAgreement-2 vs AlignAtt/SimulStreaming (2 парные сессии). Переключение дефолта на AlignAtt снизило worst-observed задержку коммита с ~23 с до <1 с, а зависания декодера (15 → 0 watchdog-сбросов) и галлюцинации (16 → 0) ушли за сессию. Числа логированы и воспроизводимы скриптом сравнения A/B.
- **Диагностика холодного старта.** Свёл галлюцинации первой сессии — спайк задержки ~20 с и мусорный транскрипт на 1476 символов — к прогреву модели, который зависел от сетевого ассета и молча таймаутил; починил, упаковав прогрев локально.
- **Честные метрики задержки.** Обнаружил, что перцентили задержки схлопываются в ноль из-за паттерна коммитов нового режима, поэтому средние прячут реальные спайки; вместо них добавил проверки worst-case-задержки и доли спайков как pre-ship gates.

## Метрики (snapshot)

Замеры на парных ~1-часовых сессиях, одна машина (Apple Silicon, small.en); внутренний A/B, не multi-condition бенчмарк. Полный отчёт: [A/B benchmark](/evidence/consul-ab-benchmark.html).

| Метрика | Результат |
|---|---|
| Worst-observed задержка коммита STT (A/B алгоритмов) | 23 с → <1 с |
| Зависания декодера (watchdog resets) / сессия | 15 → 0 |
| Сброшенные галлюцинированные коммиты / сессия | 16 → 0 |
| Ошибки пайплайна / сессия | 24 → 0 |
| Спайк задержки на холодном старте (фикс прогрева) | 20.2 с → устранён |
| STT-движков за одним fallback-роутером | 5 (WhisperLiveKit, whisper.cpp, Apple Speech, Deepgram, AssemblyAI) |
| LLM-провайдеров (1 локальный + 3 облачных fallback) | 4 (Ollama · OpenAI · Anthropic · Google) |
| Автотестов | 638 проходят |
| Байт документа через IPC main→renderer | 0 (только booleans — privacy-инвариант) |

## Архитектура

![Architecture diagram](../docs/images/architecture.svg)

AI Consul работает как multi-window Electron-приложение. Renderer отвечает за пользовательские окна и захват аудио в браузерном слое; main process управляет оркестрацией сессии и IPC. Слой транскрибации сначала пробует streaming-путь и переходит на batch-обработку, если live-маршрут недоступен. Вывод транскрипта объединяется с контекстом и prompt-логикой, после чего подсказки отправляются в UI.

## Routing и fallback-сценарии

![Pipeline and fallback diagram](../docs/images/pipeline.svg)

Ключевая идея — не один конкретный model/backend, а управляемая стратегия обработки:

- Сначала пробовать лучший доступный путь.
- Переходить на fallback без разрушения user experience.
- Делать local/cloud trade-offs видимыми, а не скрытыми.
- Проектировать graceful degradation как часть продукта.

## Ретро / Lessons Learned

- **Что сработало:** относиться к производительности как к эксперименту — воспроизводимый A/B-харнесс + инструментирование каждого коммита превратили расплывчатое «тормозит» в измеренный выигрыш по worst-observed-задержке (с ~23 с до <1 с); смена алгоритма стала решением по данным, а не догадкой.
- **Что не сработало:** заглушки и зависящие от среды куски, падавшие молча — stub-суммаризатор привязывал подсказки к устаревшему контексту, а прогрев модели зависел от сетевого запроса и таймаутил у пользователя (~20 с задержки первой сессии + галлюцинации).
- **Что бы изменил:** инструментировать задержку и упаковывать/проверять runtime-зависимости с первого дня; проверять методологию метрик до того, как им доверять — перцентили показывали «0 мс» и прятали реальные спайки, пока не добавил monotonic worst-case + spike-rate гейты.

- Репозиторий приватный.
